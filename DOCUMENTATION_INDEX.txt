🔑 KEY TAKEAWAYS
═════════════════════════════════════════════════════════════════════════════

ARCHITECTURE:
• React frontend (localhost:5173) ↔ FastAPI backend (localhost:8000)
• Gemini AI for story and audio generation (frontend)
• SQLite database for persistence
• Audio files stored in /backend/uploads/ directory

STORY GENERATION:
• User enters vocabulary words
• Frontend calls Gemini API (gemini-3.1-flash-lite-preview)
• Gemini returns JSON: {title, story with highlights, vocabulary}
• Frontend displays story with highlighted vocabulary words

STORY SAVING:
• User clicks "Save"
• Frontend generates audio (optional) via Gemini TTS
• Audio converted to base64
• POST request to /stories with all data
• Backend saves audio file to /uploads/, metadata to database

VOCABULARY TABLE:
• Generated by Gemini AI for each word
• Structure: word, meaning_in_target, equivalent_in_english
• Used as learning aid with mnemonics
• Displayed in interactive table

AUDIO:
• Generated by Gemini TTS (gemini-2.5-flash-preview-tts)
• Voice: "Leda"
• Format: Base64 PCM or standard audio format
• Playback: HTML audio element with custom controls
• Sync: Click word → jumps to that word's time in audio

AUTHENTICATION:
• All endpoints require: Header "X-API-Key: {WHISPER_API_KEY}"
• Validated against config.WHISPER_API_KEY
• Set via environment variable

QUIZ FEATURE:
• NOT IMPLEMENTED
• Feature checklist marks as incomplete
